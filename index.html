iMac访问局域网

发表于 2018-03-28

iMac插好网线，访问局域网设置

$ networksetup -setv6off Ethernet
$ networksetup -setmanual Ethernet 192.168.188.111 255.255.255.0 192.168.188.1

其中第一个192.168后面的188要和被访问的局域网的一致（我访问的是：http://192.168.188.11:8088），后一个192.168后面的188可以更改，如果仍然无效，将111更改为其它再试

Hexo的使用

发表于 2018-03-24 | 分类于 iOS
  • 如果不是首次写GitHub博客，比如换电脑的情况，在source/_posts目录下书写新的.md文件，执行以下即可
  • $ cd wangtaoios.github.io
$ hexo clean    //清除缓存
$ hexo g        //生成本地发布文件
$ hexo d        //发布
//之后要输入账号（wangtaoios）和密码

    hexo命令

    wangtaoios.github.io

一：安装node.js

1：下载node.js

2：检验安装成功：

$ node -v

二：安装hexo

$ sudo npm install hexo-cli -g

三：运行（已在GitHub创建github.io的public仓库）

$ hexo init wangtaoios.github.io
$ cd wangtaoios.github.io
$ sudo npm install
$ hexo clean
$ hexo g
$ hexo s
$ sudo npm install hexo-deployer-git --save
$ vim _config.yml

然后在_config.yml最后4行配置如下，其中最后3行要首行缩进2个字符，“:”后面都要空一个字符

deploy:
  type: git
  repo: https://github.com/wangtaoios/wangtaoios.github.io.git
  branch: master

最后执行

$ wq
$ hexo clean
$ hexo g
$ hexo d

-ObjC 和 -all_load 的使用

发表于 2018-03-22 | 分类于 文档

APP版本升级的时候遇到的BUG

2017年5月份上线时候，APP用的乐视云直播SDK4.5（2017.1.05），2017下半年时候发现，iOS11系统，播放有声音，无画面

APP新版本的时候就使用最新版本乐视云直播SDK4.6（2017.5.12），但是在SDK4.5里要求

当我换为新SDK后，报错

1：查看 Build Settings -> Library Search Paths
里面SDK路径没问题

2：Build Settings —> Other Linker Flags 里包含有
-ObjC 和 -all_load

最后将-all_load删掉，就可以了

  • Library Search Paths：

    附加到项目中的第三方Library(.a files)的搜索路径，Xcode会自动设置拖拽到Xcode中的.a文件的路径，为了便于移植或者是多人协作开发一般会手动设置。

  • Other Linker Flags

    －ObjC：加了这个参数后，链接器就会把静态库中所有的Objective-C类和分类都加载到最后的可执行文件中。
－all_load：会让链接器把所有找到的目标文件都加载到可执行文件中，但是千万不要随便使用这个参数！假如你使用了不止一个静态库文件，然后又使用了这个参数，那么你很有可能会遇到ld: duplicate symbol错误，因为不同的库文件里面可能会有相同的目标文件，所以建议在遇到-ObjC失效的情况下使用-force_load参数。
-force_load：所做的事情跟-all_load其实是一样的，但是-force_load需要指定要进行全部加载的库文件的路径，这样的话，你就只是完全加载了一个库文件，不影响其余库文件的按需加载。

iOS使用YYKit实现表情键盘

发表于 2018-03-21 | 分类于 iOS

项目需求，APP发布内容的时候支持表情

需求：

点击键盘的表情按钮，比如[微笑]，在输入框显示🙂，发布成功到服务器时候，服务器存[微笑]；请求下来数据在APP展示的时候，仍然显示🙂。

原理：

点击键盘的表情按钮，比如[微笑]，此时将字符串[微笑]传到输入框，输入框使用YYTextView，设置YYTextView的textParser即可展示对应表情🙂。

关于textParser：

在YYTextSimpleEmoticonParser发现

  • parser.emoticonMapper 是字典，其中键：NSString；值：UIImage。
  • 本地有个plist文件，也就是字典，其中键：NSString，表情的汉字版（如：[微笑]）；值：NSString，图片名字。
  • 那么由本地plis文件生成对应的parser.emoticonMapper字典就可以设置YYTextView的textParser。
关于键盘的自定义：

点击切换按钮的时候，来切换键盘

WTEmoticonInputView即为自定义的表情键盘页面，整体是个UIScrollView，每一页表情是一个UIView，根据表情数量将这些UIView放到UIScrollView。设置相应的代理，将点击的表情按钮对应的字符串传到_textView即可。

同理，从服务器请求的数据，展示的时候，将YYTextView换为YYLabel，设置相应的textParser即可。

中华人民共和国最高人民法院之重要司法解释

发表于 2018-03-21 | 分类于 文档
1：最高人民法院关于修改关于适用《中华人民共和国公司法》若干问题的规定的决定

2014-02-27
http://www.court.gov.cn/fabu-xiangqing-6135.html

第十三条 股东未履行或者未全面履行出资义务，公司或者其他股东请求其向公司依法全面履行出资义务的，人民法院应予支持。
公司债权人请求未履行或者未全面履行出资义务的股东在未出资本息范围内对公司债务不能清偿的部分承担补充赔偿责任的，人民法院应予支持；未履行或者未全面履行出资义务的股东已经承担上述责任，其他债权人提出相同请求的，人民法院不予支持。
股东在公司设立时未履行或者未全面履行出资义务，依照本条第一款或者第二款提起诉讼的原告，请求公司的发起人与被告股东承担连带责任的，人民法院应予支持；公司的发起人承担责任后，可以向被告股东追偿。
股东在公司增资时未履行或者未全面履行出资义务，依照本条第一款或者第二款提起诉讼的原告，请求未尽公司法第一百四十七条第一款规定的义务而使出资未缴足的董事、高级管理人员承担相应责任的，人民法院应予支持；董事、高级管理人员承担责任后，可以向被告股东追偿。
